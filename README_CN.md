# ChatMap

*一张给长对话用的可视化地图。*

[English](README.md)

## 为什么需要它？

跟AI聊天很少是一条直线。

你问了一个问题，岔出一个相关的问题，再岔出另一个。几分钟后你想接着原来的话题往下问，却已经找不到它在哪了。

对话越长，找回之前的话题就越难。

ChatMap把你问的每个问题都变成一棵可视化的对话树。不用在几百条消息里翻找，点一下就能直接跳回任意一个之前的问题。

## 例子

原来是这样：

```text
问题A
 ↓
问题B
 ↓
问题C
 ↓
随便岔开的问题
 ↓
又一个岔开的问题
 ↓
我刚才问到哪了？
```

现在是这样：

```text
问题A
├─ 问题B
├─ 问题C
│  ├─ 岔开的问题
│  └─ 又一个岔开的问题
└─ 下一个主线话题
```

点地图上的任意一个节点，直接跳回对话里那个位置。

## 使用场景

### 研究与写作
主线问题一直往下推进，中途岔开去聊细节也不会丢掉原来的脉络。

### 调试代码
查相关API或实现细节，不会忘了原来在排查哪个bug。

### 做决定
对比几个选项，深入聊某一个的细节，随时能立刻跳回大局。

### 长期使用的AI对话
把ChatGPT、Claude或Gemini当长期助手用，不会被巨大的聊天记录淹没。

### 学习
跟着一个主题往下学，中途问点临时想到的问题，再接着回到原来的地方。

## 功能

### 自动记录
你发出去的每个问题都会自动加到地图上。

### 主线与分支
接着主线往下聊，或者为岔开的问题临时开一条分支。

### 一键跳转
点一下，立刻跳回任意一个之前的问题。

### 手动节点
随时手动把重要的消息加进地图。

### 编辑工具
- 节点改名
- 删除节点
- 撤销 / 重做
- 折叠分支

### 双语界面
安装的时候选中文还是English。

### 支持的平台
- ChatGPT
- Claude
- Gemini

以后可能会支持更多平台。

## 安装

### 开发者模式
1. 打开 `chrome://extensions`
2. 打开"开发者模式"
3. 点"加载已解压的扩展程序"
4. 选中这个项目文件夹
5. 选择你想用的语言
6. 打开一个支持的AI对话页面
7. 点插件图标

## 已知限制
- 自动记录目前只能识别"按Enter发送"这种方式
- 刷新页面之后，较早的节点跳转准确度会下降
- 撤销/重做记录只在当前这次浏览器会话里有效
- 语言目前只能在安装时选择一次

## 路线图
